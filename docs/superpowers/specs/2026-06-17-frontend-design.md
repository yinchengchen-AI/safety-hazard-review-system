# 安全生产隐患复核系统 — 前端页面设计与美化方案

> 状态：已确认
> 日期：2026-06-17
> 对应系统文档：2026-06-17-safety-hazard-review-system-design.md

## 1. 设计目标

在现有功能基础上，为「安全生产隐患复核系统」建立统一、专业、可维护的前端视觉体系，并逐一美化所有页面。目标用户为应急管理局监管员、科长、局长及系统管理员，使用场景兼顾办公室桌面端与现场手机端（PWA）。

## 2. 范围

- 全局设计系统（颜色、字体、间距、组件变体、暗色模式）
- 全部页面视觉与交互升级：
  - 登录页
  - 工作台（首页）
  - 案件列表页
  - 案件详情页
  - 登记案件 / 批量导入 / 复核 / 审核表单页
  - 统计页
  - 通知中心
  - 同步状态页
  - 管理后台（用户、企业、隐患类型、模板、审计日志）
- 响应式布局：桌面端 + 移动端（PWA 现场使用）
- 不改动业务逻辑、数据模型、API 接口与状态机

## 3. 设计决策记录

| #   | 决策         | 选择                                      | 备选                 | 理由                                       |
| --- | ------------ | ----------------------------------------- | -------------------- | ------------------------------------------ |
| 1   | 整体风格     | 政务蓝 · 稳重专业                         | 现代 SaaS / 深色大屏 | 贴合政府监管系统调性                       |
| 2   | 暗色模式     | 需要                                      | 仅浅色               | 支持夜间值班与长时间办公                   |
| 3   | 桌面导航     | 顶部导航栏                                | 左侧边栏             | 顶部导航对桌面更简洁，信息区更宽           |
| 4   | 移动端导航   | 底部 Tab                                  | 侧滑抽屉             | 现场频繁切换场景更适合底部 Tab             |
| 5   | 登录页品牌   | 文字标题，无图形 Logo                     | 图形 Logo / 图标     | 更庄重、不引入额外设计资产                 |
| 6   | 工作台增强   | KPI + 待办 + 趋势图 + 通知摘要 + 快捷入口 | 仅 KPI + 待办        | 满足用户「关键指标、角色化入口、动态」需求 |
| 7   | 案件列表增强 | 筛选 + 搜索 + 排序 + 分页                 | 仅美化表格           | 提升中大列表的可用性                       |
| 8   | 统计图表     | 引入 Recharts                             | 纯 CSS 图表 / 文本   | 趋势与分布需要专业图表表达                 |
| 9   | 表单布局     | 分组卡片                                  | 长表单单列           | 降低认知负担，字段关系清晰                 |
| 10  | 实施路径     | 渐进式改造                                | 全局换肤一次性重构   | 风险低，可逐页验证效果                     |

## 4. 设计系统

### 4.1 色彩

采用 CSS 变量管理，支持 light/dark 切换。政务蓝为主色，状态色语义统一。

#### 浅色模式

- 主色：`#1e40af`（政务蓝）
- 强调色：`#3b82f6`
- 浅蓝背景：`#dbeafe`
- 背景：`#ffffff` / `#f8fafc`
- 文字：`#1e293b`（主文）/ `#64748b`（辅助）
- 边框：`#e2e8f0`
- 成功：`#16a34a` / 警告：`#f59e0b` / 错误：`#dc2626`

#### 暗色模式

- 背景：`#0f172a`
- 卡片：`#1e293b`
- 强调：`#60a5fa`
- 文字：`#f8fafc` / `#94a3b8`
- 边框：`#334155`

#### 状态色

| 状态   | 背景      | 文字      | 用途     |
| ------ | --------- | --------- | -------- |
| 待复核 | `#fef3c7` | `#92400e` | 案件状态 |
| 待审核 | `#dbeafe` | `#1e40af` | 案件状态 |
| 审核中 | `#ede9fe` | `#6d28d9` | 案件状态 |
| 已销案 | `#dcfce7` | `#166534` | 案件状态 |
| 重大   | `#fee2e2` | `#991b1b` | 严重度   |
| 较大   | `#ffedd5` | `#9a3412` | 严重度   |
| 一般   | `#dcfce7` | `#166534` | 严重度   |

### 4.2 字体

- 字体栈：系统无衬线（`PingFang SC`, `Microsoft YaHei`, `Inter`, sans-serif）
- 页面标题：20px / font-weight 600
- 卡片标题：14px / font-weight 600
- 正文：14px / font-weight 400
- 辅助文字：12–13px / color muted
- 工单号：等宽字体 `ui-monospace`, `SFMono-Regular`, monospace

### 4.3 圆角与阴影

- 卡片圆角：12px
- 按钮圆角：8px
- 输入框圆角：8px
- 徽章圆角：9999px
- 卡片阴影：`0 1px 3px rgba(0, 0, 0, 0.06)`，hover 时加深
- 登录卡片阴影：`0 4px 24px rgba(0, 0, 0, 0.06)`

### 4.4 间距

- 页面内边距：桌面 24px，移动端 16px
- 卡片内边距：16–20px
- 卡片间距：12–16px
- 表单字段间距：14–16px
- 最大内容宽度：1400px，水平居中

### 4.5 组件变体

基于 shadcn/ui 默认组件进行样式覆盖：

- **Button**
  - Primary：背景 `#1e40af`，文字白色
  - Secondary：背景 white，边框 `#e2e8f0`，文字 `#1e293b`
  - Danger：背景 `#dc2626`，文字白色
- **Input / Textarea / Select**
  - 高度 40–44px，圆角 8px
  - 聚焦 ring：`2px #3b82f6`
- **Card**
  - 圆角 12px，白色/深色背景，浅阴影
- **Badge**
  - 圆角 9999px，按语义使用状态色
- **Table**
  - 表头 `#f8fafc` 背景，行 hover 高亮

## 5. 全局布局

### 5.1 桌面端

- 顶部固定导航栏：品牌标题 + 主导航 + 用户信息 + 登出
- 导航项：工作台、案件、统计、通知，管理员额外显示「管理」
- 内容区：最大宽度 1400px，居中，顶部留出导航高度

### 5.2 移动端

- 顶部只保留品牌标题（可选）
- 底部固定 Tab：首页、案件、统计、通知
- 内容区底部增加 Tab 高度留白

### 5.3 暗色模式

- 导航栏在暗色模式下保持深蓝 `#1e40af` 以强化品牌识别
- 内容区切换为深色背景与深色卡片

## 6. 页面设计

### 6.1 登录页

- 全屏居中卡片，浅色背景 `#f8fafc`，暗色背景 `#0f172a`
- 标题：「安全生产隐患复核系统」，加粗，字距略宽
- 副标题：「请登录后继续」
- 表单：邮箱、密码、登录按钮
- 错误提示：红色文字，位于按钮上方
- 无图形 Logo

### 6.2 工作台

- 顶部：页面标题 + 快捷操作（登记案件、批量导入）
- KPI 卡片：4 列网格（总案件、待处理、审核中、已销案），左侧彩色边框区分语义
- 下方 3 列网格：
  - 待办任务（案件号 + 状态 + 紧急程度 + 处理入口）
  - 近 7 天趋势（Recharts 迷你柱状/折线图）
  - 通知摘要（临期 / 审核 / 同步分类）
- 移动端：KPI 2 列，下方模块上下堆叠

### 6.3 案件列表页

- 顶部：标题 + 批量导入/登记案件按钮
- 筛选栏：搜索框（企业/案件号）、状态下拉、隐患类型下拉、严重度下拉、排序选择
- 表格列：工单号、企业、隐患类型、状态、严重度、登记时间、操作
- 状态徽章与严重度颜色按设计系统
- 分页：每页 20 条，页码 + 上下页
- 移动端：表格改为卡片列表

### 6.4 案件详情页

- 页头：工单号 + 状态徽章 + 严重度徽章 + 主要操作按钮
- 桌面端 2 列布局：
  - 左侧：基本信息卡片 + 复核/审核时间线
  - 右侧：附件照片网格 + 案件状态摘要
- 时间线：垂直时间轴，当前节点高亮
- 操作按钮按状态动态显示：待复核→开始复核，待审核/审核中→审核，已销案→归档信息
- 移动端：单列堆叠，时间线简化为卡片列表

### 6.5 表单页（登记 / 批量导入 / 复核 / 审核）

- 使用分组卡片组织字段：
  - 登记案件：企业信息、隐患信息
  - 复核：Checklist 逐项结果、照片上传、备注
  - 审核：结论选择、签字、意见
- 必填项标 `*`
- 输入框统一高度与圆角
- 底部操作区：取消 + 提交
- 移动端单列，底部固定提交按钮

### 6.6 统计页

- KPI 卡片：总案件、销案率、平均处理时长、临期案件
- 近 30 天趋势图（Recharts 折线/柱状图）
- 隐患类型分布（水平条形图）
- 状态分布（彩色色块卡片）
- 移动端图表上下堆叠

### 6.7 通知中心

- 列表展示，未读用白色卡片 + 彩色左侧边线，已读用浅灰背景
- 每条显示：标题、时间、摘要
- 支持标记已读/全部已读

### 6.8 同步状态页

- 在线状态灯 + 最后同步时间
- 同步队列摘要：已同步 / 待同步 / 失败数量
- 失败项支持重试

### 6.9 管理后台

- 沿用案件列表的表格风格
- 用户管理、企业管理、隐患类型管理、模板管理、审计日志
- 操作按钮简洁（编辑、启用/禁用、删除）

## 7. 响应式策略

| 断点 | 宽度       | 布局调整                                 |
| ---- | ---------- | ---------------------------------------- |
| sm   | < 640px    | 单列布局，底部 Tab，KPI 2 列，表格变卡片 |
| md   | 640–1024px | 2 列布局，顶部导航保持                   |
| lg   | > 1024px   | 完整桌面布局，4 列 KPI，案件详情左右布局 |

## 8. 图标与图形

- 图标库：Lucide React（已在项目中使用）
- 不使用自定义图形 Logo
- 趋势图/分布图使用 Recharts

## 9. 动画与交互

- 页面切换：保持 Next.js 默认行为，不额外添加复杂转场
- 按钮 hover：背景色加深或轻微上移
- 卡片 hover：阴影加深
- 加载状态：按钮内 spinner，页面骨架屏（可选）
- Toast 提示：使用现有 shadcn/ui Toast 组件，统一成功/错误/警告样式

## 10. 异常与空状态

- 空列表：居中图标 + 「暂无数据」+ 可选操作按钮
- 加载中：骨架屏或 spinner
- 错误页：简洁错误提示 + 返回首页
- 无网络（PWA）：提示离线状态，显示本地缓存数据

## 11. 实现建议

### 11.1 第一阶段：设计系统

1. 更新 `src/app/globals.css`，定义 light/dark CSS 变量
2. 覆盖 shadcn/ui 组件样式（button、input、card、badge、table 等）
3. 添加暗色模式切换（可基于 `next-themes` 或系统偏好）
4. 更新 `tailwind.config.ts` 扩展颜色与断点

### 11.2 第二阶段：全局布局

1. 改造 `src/app/layout.tsx` 顶部导航
2. 新增移动端底部导航组件
3. 更新页面外层容器与间距

### 11.3 第三阶段：逐页改造

按优先级逐个页面改造：

1. 登录页
2. 工作台
3. 案件列表 / 详情
4. 统计页
5. 表单页（登记 / 复核 / 审核）
6. 通知 / 同步 / 管理页

### 11.4 第四阶段：组件抽象

高频业务组件抽象为可复用组件：

- `PageShell`：页面标题 + 操作区 + 内容区
- `KpiCard` / `KpiGrid`
- `StatusBadge` / `SeverityBadge`
- `FilterBar`
- `DataTable`（支持移动端卡片化）
- `Timeline`
- `ChartCard`
- `EmptyState`

### 11.5 依赖

- `recharts`：统计图表
- `lucide-react`：图标（已存在）
- 可选 `next-themes`：暗色模式切换

## 12. 测试与验收

- 视觉验收：所有页面在 light/dark 模式下无样式问题
- 响应式验收：桌面端、平板端、手机端布局正常
- 功能验收：现有 E2E 测试（登录、复核、审核、离线同步）通过
- 性能验收：图表按需加载，移动端首屏无卡顿
